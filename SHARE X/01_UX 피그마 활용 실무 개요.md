# UX 피그마 활용 실무 개요

## 1. 강의 소개
- UXUI 통합 프로세스 : UX Process (화면 설계) + UI Process (화면 디자인)
- UI와 별개로 UX 설계 업무를 주로 하거나 or 피그마 툴을 활용한 화면 설계를 작업하는 실무자 대상

## 2. UX 화면 설계서(Story Board) 핵심 구조
### [개념 정의]
- **Story Board(SB):** 화면 설계, 정책, 프로세스, 콘텐츠 구성 등 구현에 필요한 모든 정보를 담은 문서

### [문서 필수 구성 요소]
- **표지:** 문서 제목 및 프로젝트 정보 명시
- **히스토리:** 버전별 수정 이력 및 변경 사항 기록 (v0.1, v0.8 등)
- **중간 인트로:** 파트별 구분을 위한 장표 삽입
- **공통 가이드:** 전체 구조, 플로우, 공통 UI 컴포넌트(입력 필드 등) 정의
- **화면 정의 장표:** 좌측 와이어프레임과 우측 상세 설명(Description)의 매칭 구조
  - **와이어프레임(Wireframe):** 선(Wire)과 틀(Frame)을 이용하여 화면의 구조를 잡는 행위
- **마무리:** 문서의 종결을 알리는 엔딩 장표

## 3. 효율적인 협업 및 프로젝트 관리
### [메뉴 단위 분할 작업]
- **효율성 제고:** 대규모 프로젝트 시 메뉴 단위(공통, 홈, 마이페이지 등)로 파일을 분리하여 동시 작업 수행
  - 예: 01. 회원가입/로그인_SB_ver0.8_20240609 02. 홈 03. 카테고리 04. 검색 05. 상품 상세 06. 장바구니 07. 주문/결제 08. 이벤트/기획전 09. 좋아요/최근 본 10. 마이페이지 11. 설정
- **00. COMMON GUIDE** : 서비스 전반에서 공통으로 정의돼야 하는 것.
  - 예: 기본 화면 구조, 스크롤 시 작동, 다양한 입력필드 케이스

### [실무 협업 가이드]
- **커뮤니케이션 강화:** UI 디자이너, 퍼블리셔, 개발자 간의 일관된 산출물 공유
- **리스크 방지:** 불필요한 재작업 방지, 일정 차질 및 비용 낭비 최소화
- **통합 Align:** 모든 담당자의 고민이 반영된 정의 문서를 통해 프로젝트 완성도 향상

## 4. UX/UI 통합 프로세스 및 에이전시 실무
### [전략 및 분석 단계]
- **고객사 파트너사 의뢰:** RFP 전달, 계약 진행 및 요구사항 상세 정리
  - **RFP**: Request For Proposal. 제안 요청서.
- **리서치 및 분석:** 데스크 리서치. 고객사/유사 서비스 분석, 레퍼런스 및 시장 조사 실시
- **인터뷰 수행:** 임직원 및 사용자 인터뷰를 통한 인사이트 도출
- **방향성 도출:** 사용성 분석, 목업 아이데이션 및 디자인 방향성 제안

### [설계 및 구축 단계]
- **UX/UI 전략 수립:** 서비스 방향 설정 및 사용자 시나리오 제작
- **시안 제작:** 주요 페이지 중심의 UI 디자인 시안 작업
- **화면 설계:** 피그마를 활용한 UX 화면 설계서(SB) 작성
- **UI 디자인:** UI 디자인 페이지 제작
- **서비스 구현:** 퍼블리싱 및 개발 단계에서의 밀착 협업 수행

### [검증 및 오픈 단계]
- **품질 관리:** QA 테스트를 통한 오류 수정 및 기능 검증
- **오픈 준비:** 최종 런칭을 위한 운영 환경 점검 및 서비스 오픈
