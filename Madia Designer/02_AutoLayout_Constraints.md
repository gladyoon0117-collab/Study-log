# 02_AutoLayout_Constraints

## 1. Auto Layout 기초
- **Gap**: 요소 사이의 간격(Margin). AL 패널에서 수치를 직접 수정하거나 캔버스에서 클릭하여 조정 가능.
- **Direction (Flow)**: 요소가 쌓이는 방향 설정 (가로/세로).
- **생성 및 해제**: 
  - 생성: 요소들을 그룹화(`Cmd` + `G`)한 뒤 AL 적용 가능.
  - 해제: `Cmd` + `Shift` + `G` 또는 우클릭 후 `Ungroup`.
- **구성 요소 활용**: 별도의 배경 도형 없이 AL 프레임 자체에 `Padding`(내부 여백), `Fill`(배경색), `Stroke`(테두리), `Radius`(곡률)를 적용해 버튼이나 박스 제작 가능.
- **Alignment (정렬)**: 총 9개의 포인트(상/중/하 × 왼/중/오)를 기준으로 정렬. 부모 프레임의 전체 크기가 기준이 됨.
- **장점**: 디자인 수정이 빈번할 때 요소를 삭제/추가해도 간격이 자동으로 유지되어 효율적임.

## 2. 레이어 계층 구조와 실습 (Hierarchy)
- **중첩 구조 (Nested AL)**: 간격이 서로 다른 요소들을 배치할 때 유용함.
  - 예: 본문 박스 4개를 먼저 AL로 묶고, 그 그룹과 타이틀을 다시 한번 AL로 묶어 상하 간격을 다르게 설정.
- **실습 프로세스**:
  1. `이름` + `내용` 그룹핑
  2. `그룹` + `이미지`를 AL로 묶기 (Padding 설정)
  3. 완성된 박스 4개를 다시 AL로 묶기
  4. 묶인 박스 그룹과 `타이틀`을 최종 AL로 묶어 전체 구조 완성.

## 3. Resizing: Hug / Fixed / Fill
객체와 텍스트 박스의 유동적인 크기 변화를 결정하는 핵심 기능.

| 구분 | 개념 | 특징 |
| :--- | :--- | :--- |
| **Hug (Contents)** | "내가 안쪽 요소에 맞출게" | 내부 텍스트나 요소가 길어지면 프레임도 같이 커짐 (말풍선, 버튼에 유리). |
| **Fixed (Size)** | "나는 이 크기로 고정할게" | 내부 요소와 상관없이 지정된 수치(px)로 크기가 고정됨. |
| **Fill (Container)** | "내가 바깥 부모 크기에 맞출게" | 부모 프레임의 남는 공간을 가득 채움. 반응형 디자인(말줄임표 처리 등)에 유리. |

- **Clip Content**: 내용물이 프레임보다 클 때 잘라내서 보여주는 기능 (좌우 스와이프 카드 UI 디자인 등에 활용).
- **주의사항**: 가장 바깥쪽 최상위 프레임에는 `Fill` 옵션이 나타나지 않음 (채울 '부모'가 없기 때문). 자식 프레임부터 설정 가능.

## 4. Constraints
해상도가 변할 때 내부 요소가 어떻게 반응할지 결정하는 지능형 컨테이너 기능.

- **기본 원리**: 프레임 크기가 변할 때 내부 요소를 어디에 고정할지 결정. (반응형 웹/앱 기획 시 필수)
- **방향 설정**: 
  - **Left / Right / Center**: 가로축 고정 위치 결정.
  - **Top / Bottom / Center**: 세로축 고정 위치 결정.
  - **Left & Right (양방향 고정)**: 프레임이 늘어날 때 양옆 마진(Margin)을 유지하며 요소의 가로 길이가 함께 늘어남.
- **텍스트와 Constraints**: 
  - `Left & Right` 설정 후 타이포그래피 설정에서 **말줄임표(...)**를 활성화하면, 프레임이 좁아질 때 글자가 잘리지 않고 말줄임표로 깔끔하게 처리됨.
- **작동 조건**: 
  - Constraints는 부모 프레임 바로 아래에 있는 **직계 자식 요소**에게만 활성화됨.
  - `Option`을 누른 채 요소를 드래그하면 텍스트 상자 등을 양방향으로 동시에 늘릴 수 있음.

## 5. 실무 팁 (Tips)
- **개별 마진 수정**: 전체를 AL로 묶었으나 특정 요소만 마진을 다르게 주고 싶다면, 그 요소만 따로 한 번 더 AL로 감싼 뒤 값을 수정함.
- **반응형 구현 방식**: 
  1. 오토 레이아웃의 `Gap = Auto`를 활용하거나, 
  2. `Constraints`를 활용해 고정 위치를 지정하는 방식 중 상황에 맞는 방법을 선택함.

# auto layout
- gap : 요소 간 margin. 고정값. margin 클릭하면 뜨는 수치 변경하거나, 우측 AL 패널에서 값 수정해서 값 변경할 수 있음.
- AL 패널 내 flow : 가로 방향으로 묶이느냐, 세로 방향으로 묶이느냐
- background+요소들 그룹(command+G) 해서 그룹화된 거 그대로 AL 걸어도 됨.
- command+shift+G or 우클릭 후 ungroup : 그룹 해제
- 내용 AL 걸고 padding+fill+round = background 별도로 쓰지 않고 만들 수 있음
  - padding : AL 걸린 요소 한 그룹의 양옆 여백 값, 상하 여백 값
  - fill : 그 요소 뒤에 깔리는 색. text 잡고 fill 누르면 배경 채워지듯.
- alignment : 수평 정렬(왼/중/오)와 수직 정렬(상/중/하)로 총 9개로 나뉨. 오토 레이아웃의 부모 프레임의 전체 높이와 너비가 기준.
- 디자인은 수정이 빈번한데 AL은 어떤 요소를 지워도 자동으로 따라붙게 하는 등 전체적으로 페이지 수정할 때 용이함
- bullet형 만들 때, bullet 기호랑 텍스트 따로 적어서 AL 걸고 alignment 왼쪽상단 정렬.

# 실습
- 타이틀과 본문과의 간격, 본문 4개 박스 간의 간격을 다르게 설정하기 위해서는 -> 본문 박스 4개끼리 AL 걸고 타이틀과 그 AL된 그룹을 또 한 번 AL. 박스를 개별적으로 타이틀과 같이 하면 박스간의 간격이 곧 타이틀과 박스의 간격이 됨.
- 이름+내용 그룹 -> 그룹과 이미지 AL -> 상하좌우 여백 20씩 -> 그렇게 완성된 박스 4개를 묶어 AL-> 그렇게 묶인 박스 4개 AL을 또 한번 타이틀과 AL-> 양옆 마진값 20 -> 마지막으로 한 번 더 전체 AL-> 구조 완성

# horizontal resizing
- clip content : 좌우 스와이프 디자인할 때 좌우측 크기 줄이면 작아지는 게 아니라 잘린 듯한 느낌. e.g. 브라우저 크기에 다 안 담길 만큼 객실 카드 있는 경우에 끝처리
- hug/fixed/fill : 가로와 세로 개별 설정 / rectangle에 거는 것과 텍스트 박스에 거는 것 별개. 박스에는 fix, 텍스트 박스에는 fill 되어있으면 박스는 텍스트 늘어나면 박스는 고정, 텍스트 박스만 늘어나는 상황. 둘 다 가로세로 hug로 설정해줘야 다같이 움직임.
  - hug : 메신저나 텍스트 디자인할 때 텍스트가 길어지면 말풍선도 같이 길어져야 함. 텍스트 길이나 줄바꿈에 따라 크기가 달라짐.
  - fixed : 고정돼서 길어져도 확장되지 않음. 내가 지정한 수치(px)로 고정하는 방식.
  - fill : 박스를 유동적으로 조절할 때 박스 크기에 맞춰서 텍스트 영역 자체가 좁아지거나 넓어짐.
  - 가장 바깥 박스에는 fill이 노출되지 않음. 자식 프레임에 fill 걸 수 있음.
  - hug와 fill의 차이 (hug와 fill이 비슷해보이면서도 정반대 개념)
    - hug : 내가 안쪽에 맞출게
    - fill : 내가 바깥쪽에 맞출게
  - 말풍선은 fill보다 hug가 더 유리, 반응형 디자인에는 fill이 유리(말줄임표)

  - e.g. box 내 텍스트는 좌측 고정, drop 버튼은 우측 고정해서 박스 줄어들면 내용물이 고정된 채로 반응하도록 (꼭 오토 레이아웃 gap=auto로 하지 않아도 됨)

- 좌+우를 같이 고정하면 양옆에 margin이 걸린 상태에서 반응.
- 상단을 고정하면 박스를 상하로 늘렸을 때 내용물이 상단에 고정된 채로 늘어남. center로 고정하면 frame의 가로/세로의 중앙을 지키며 늘어남.
- 텍스트 영역을 option 누르고 drag하면 상자가 양쪽으로 늘어남.
- 텍스트 상자의 경우, left+right 고정하고 typography에서 말줄임표 설정해주면 텍스트가 frame 안에서 양쪽이 고정된 채로 자리를 지키면서 줄어드는 영역만큼 말줄임표.
  - 너무 빨리 말줄임표 처리되는 건 텍스트 상자 양옆의 margin 간격 때문. 방지하고 싶으면 텍스트 상자 자체를 넓게 잡아주면 됨.
  - left+right 설정할 때는 거의 필수적으로 말줄임표 해야 예쁨. 아니면 줄바꿈돼서 box 벗어나는 현상.
- auto layout으로 반응형 만드는 방법 : 프레임+요소까지 만들고 -> 텍스트는 fill container
- 전체 요소들 묶어서 auto layout 걸었는데 그중 일부 margin 값만 수정하고 싶을 땐, 수정하고 싶은 그 요소만 묶어서 한번 더 auto layout을 걸어준 뒤 값 바꿔주면 됨.
- contraints는 부모 프레임(frame 24)의 바로 아래 있는 자식 프레임한테만 뜸.
- auto layout+hug/fill/fixed or auto layout 없이 constraints 활용임
