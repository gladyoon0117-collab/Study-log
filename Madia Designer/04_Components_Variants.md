# 04_Component_Variants

## 1. 컴포넌트 기초
객체를 재사용 가능한 '원본'으로 만드는 기능. 메인 컴포넌트를 수정하면 복사본(Instance)이 일괄 변경됨.

- **자동 등록**: 컴포넌트를 생성하면 자동으로 `Assets` 패널에 추가되어 언제든 불러올 수 있음.
- **생성 옵션**:
  - **Create Component**: 선택한 전체를 하나의 통짜 컴포넌트로 생성.
  - **Create Multiple Components**: 2개 이상의 레이어를 각각 개별적인 컴포넌트로 한 번에 생성.

## 2. Component Set & Variants
모양은 비슷하지만 상태(State)나 설정값이 다른 요소들을 하나의 그룹으로 묶어 관리하는 기능.

- **핵심 원리**: 아이콘 색상이나 크기를 일일이 바꿀 필요 없이 우측 패널의 `Property` 변경만으로 즉시 교체 가능.
- **속성(Property) 부여 프로세스**:
  1. `Component Set` 선택 -> 우측 패널에서 속성 이름(예: 사이즈, 스타일) 입력.
  2. 각 객체를 선택해 해당 속성값(예: 56, Small, Active 등)을 입력하여 매칭.
  3. `Assets`에서 불러온 인스턴스의 우측 패널에서 설정한 속성을 선택해 자유롭게 변경.
  - *Tip: 속성 순서를 바꾸고 싶다면 Properties 패널에서 드래그하여 정렬 가능.*

모양은 비슷하지만 상태나 설정값이 다른 여러 컴포넌트를 하나로 묶어 관리함.

## 3. 고급 속성 (Advanced Properties)

### Boolean Property (불리언 토글)
- **용도**: 특정 요소(아이콘 등)를 켜거나 끄는 온/오프 스위치 제작.
- **설정**: Property 추가(`Boolean`) -> 이름 작성(예: 왼쪽 아이콘) -> 해당 요소를 선택하고 `Apply variable/property`를 통해 연결.
- **결과**: `Assets`에서 불러온 컴포넌트의 토글 스위치 하나로 요소 노출 여부 제어 가능.

### Instance Swap Property
- **용도**: 버튼 내 아이콘 등을 셀렉트 박스를 통해 다른 아이콘으로 쉽게 교체.
- **설정**: Property 추가(`Instance Swap`) -> 이름 작성 -> 기본값(Value) 설정.
- **결과**: 미리 등록해둔 아이콘 컴포넌트 세트 내의 다른 아이콘으로 실시간 교체 가능.

## 4. 실무 중요 규칙

### "아이콘을 무조건 먼저 만드세요"
- **이유**: 버튼을 다 만든 뒤에 아이콘을 컴포넌트화하면, 버튼 내의 아이콘들은 컴포넌트와 연결되지 않은 일반 레이어로 남게 됨. 이 경우 인스턴스 스왑 기능을 쓸 수 없음.
- **올바른 순서**: 
  1. 아이콘들을 먼저 제작하고 각각 컴포넌트(또는 세트)로 등록.
  2. 등록된 아이콘을 버튼 안에 넣고, 그 버튼을 다시 메인 컴포넌트로 제작.

### 메인 컴포넌트 vs 인스턴스 복사
- **메인 컴포넌트 (꽉 찬 다이아몬드 ◆)**: 원본 그 자체. 다른 메인 컴포넌트 내부에 직접 들어갈 수 없음.
- **인스턴스 (빈 다이아몬드 ◇)**: 원본의 복사본. 
- **복사 팁**: 메인 컴포넌트를 그냥 복사(Cmd+C)하는 게 아니라, `Option`을 누른 채 드래그하여 **인스턴스 상태**로 만든 뒤 다른 버튼 프레임 등에 넣어야 함.

### 속성 노출 (Expose Properties)
- **Expose Properties Item**: 버튼 내부에 들어있는 아이콘의 세부 속성(사이즈, 스타일 등)까지 상위 컴포넌트 패널에서 바로 확인하고 수정하고 싶을 때 사용함.
